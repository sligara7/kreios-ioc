Measuring the Ramping Parameter

You may want to record the ramping parameter during the ramp. There are however limitations to thisâ€”it is not possible to ramp and measure a parameter simultaneously and store the ramping parameter status with the scan. This is because the ramping parameter is a set value, while SpecsLab Prodigy is only able to record actual values. The following methods indicate ways of recording actual values while SpecsLab Prodigy is setting the ramping parameter.

Use the Live Parameter View

If you have the Live Parameter view for monitoring the status of a parameter over time, you can add the ramping parameter to the view. This will be monitored and displayed. It is also written to a file which you can open in the Data History view. This allows you to correlate the ramping parameter with the scan data.

Record a similar parameter

When you add an external input to the experiment, the parameter(s) of the external input are recorded with the scan and can be used for normalization. While you cannot ramp and include the same parameter, it is possible in some cases to record a related parameter.

 

An example of this is the temperature and emission current for sample heating. Both parameters show similar characteristics. You can use emission for the ramping parameter while monitoring the temperature as an external input.
