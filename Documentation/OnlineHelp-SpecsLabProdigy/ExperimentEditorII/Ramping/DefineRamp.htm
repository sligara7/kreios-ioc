Defining a Ramp

Before defining a ramp, you need to make sure the device is part of the experimental definition, along with the analyzer, source and any other devices you want to use.

 

To define a ramp:

1. Click the plus icon directly below the devices in the experiment and select Profiling.
2. Click Add Parameter and select the desired parameter from the list.
3. Enter the ramp settings. The diagram below shows the meaning of each setting.

 

NOTE:
It may be tempting to try to set Interval to a short time so that the steps stay closer to the linear ramp. Remember that the instrument may not be able to react to a string of commands in quick succession. In reality, the result is unlikely to be a sawtooth—the parameter value will be quasi-linear due to the lag between the applied step and the effect it has within the experiment itself.

 

 

4. Select the behavior at the end of the ramp from the Afterwards menu. You can reset the parameter value to its original value or keep the value at the end of the ramp.
5. Optionally, you can set a condition to pause the ramp depending on another parameter value. For example, you can stop the ramp when a the pressure exceeds a certain value. To set a condition:

■    Click Add Parameter and select a parameter.

■    Select a relation. For numerical parameters, you can select =,≠, ≥, ≤. For other parameters (e.g. status), you can select =,≠.

■    Enter a value for the parameter. The ramp will be paused when this condition is satisfied.

 

You can now define a spectrum group in the usual way. The ramp will run while the spectrum is measured.
