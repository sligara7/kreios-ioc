Heater Status

After connecting to the MHPS 6, the status of the pockets and pyrometer are shown. The pyrometer temperature is shown in the top line—note that it can only show readings above 750 °C; any temperature below is given as 0 °C. The pyrometer only reads the temperature of one pocket.

 

The table below lists the other features in the table.

 

Feature Description
Name The name of the pocket.
Current Filament current for the pocket heater.
Voltage Voltage of the pocket heater.
Temperature

A calculated value of the pocket temperature based on the filament current and voltage. It is important to be aware of the following:

■   The temperature is not directly measured—it is only calculated.

■   The calibration curve was obtained for one cathode type and is likely to be different with different cathodes, as the resistance–temperature characteristics vary.

■   You should under no circumstances take the reported value as being accurate—the real temperature may be considerably different.

■   The main strengths of the temperature readings are:

■    You can see when a pocket is being heated (T > 0 °C).

■    A change in the reported temperature is likely to represent a change in the real temperature. You may need to investigate the reason for this change.
Timer Countdown timer showing the amount of time left for heating. This is used with the Temperature Ramping feature.
Indicator

Indicator of the pocket status. This is color coded; a color can however have different meanings, which can be seen in a tooltip by hovering the mouse over the indicator:

■   Gray—pocket not in use.

■   White—pocket not connected.

■   Yellow—pressure too high, heater current ramping down.

■   Green (mouseover: ramping_current)—heater current ramping up.

■   Green (mouseover: constant_current)—heater running with a constant current.

■   Green (mouseover: regulating_pyro_stable)—heater regulated by the optical pyrometer.

 

You can also log the status of the heater pockets by checking the Write log file to d:\mhps.log option box. This produces a file with a timestamp. The entries in the file are delimited by a semicolon. The file has the following structure:

■    First line is the date and name.

■    Second line lists the column names.

■    All further lines contain the logged data.

 

There is a Switch Pockets Off button at the bottom of the device control that will immediately switch off the power to all pockets.
